# Design: Trinity Autonomous Choreography

This document formalizes the architecture of the **Trinity Choreography** engine, a decentralized, NLP-driven multi-agent loop designed to handle complex workflows without human intervention.

## 1. The @trinity Orchestrator Facade
To abstract the complexity of the agent hierarchy from the user, the `@trinity` persona serves as the primary entry point (Project Manager). 
- **User Experience**: The user issues a single, high-level intent to `@trinity`.
- **Delegation**: Trinity does not write code. It parses the request, formulates a technical specification, and routes it to the `@architect` to begin the loop.

## 2. Deliverables vs. Drafts
A task is not considered complete simply because code was generated. 
- **Virtual Drafts (ARTEFACT_PROPOSAL)**: Code generated by the Coder exists only as an immutable event in the VFS (`cr.sqlite`).
- **Tangible Deliverables**: For the loop to yield value, the AI must explicitly execute the code (e.g., via `@@sandbox(exec ...)`) to produce tangible artifacts (e.g., `.mp4` files) or promote the scripts physically. The Auditor verifies the `RUNTIME_OUTPUT` before declaring completion.

## 3. Declarative Orchestration (Trinity Skills)
Instead of hardcoding brittle state-machines into the CLI runtime (e.g., loop limits or forced transitions), orchestration logic is defined declaratively within **Skill** definitions (`.agents/skills/trinity_genesis/SKILL.md`).
- **The Daisy-Chain**: Agents invoke one another sequentially via NLP `@mentions`.
- **Phase Constraints**: The Architect must invoke the Coder at the end of each plan phase. The Coder must invoke the Auditor to test the code.
- **Circuit Breaker**: The Auditor enforces repetition limits strictly through conversational logic (e.g., "If 3 iterations fail, escalate to Human").

## 4. Continuous Learning Lifecycle
The Trinity facade includes a macro-level memory loop to formalize problem-solving into reusable knowledge.
- **Pre-Flight Discovery (RAG)**: Before starting the task, `@trinity` queries `.agents/skills/` and `.agents/blueprints/` for prior context, injecting recognized patterns into the Architect's briefing.
- **Post-Flight Synthesis**: Upon successful Auditor approval, `@trinity` re-activates. It analyzes the `$chatHistory` and the final artifact. If the workflow solved a novel problem, Trinity drafts a new generalized `SKILL.md` or `BLUEPRINT.md` and submits it as an `ARTEFACT_PROPOSAL` to permanently expand the repository's capabilities.
